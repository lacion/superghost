<feature>
  <name>Env Var Interpolation Engine</name>
  <files>src/config/interpolate.ts, tests/unit/config/interpolate.test.ts</files>
  <behavior>
    Core interpolation (CFG-01):
    - interpolateConfig({baseUrl: "${BASE_URL}"}, {BASE_URL: "http://localhost:3000"}) => resolved.baseUrl === "http://localhost:3000"
    - Template map records "baseUrl" => "${BASE_URL}"

    Default fallback (CFG-02):
    - interpolateConfig({baseUrl: "${BASE_URL:-http://localhost:3000}"}, {}) => resolved.baseUrl === "http://localhost:3000"
    - No error produced for unset var with default

    Required with error (CFG-03):
    - interpolateConfig({apiKey: "${API_KEY:?API_KEY must be set}"}, {}) => errors includes "API_KEY: API_KEY must be set"
    - Multiple missing required vars batched into single errors array

    Post-parse deep walk (CFG-04):
    - Walks nested objects: {tests: [{baseUrl: "${URL}"}]} resolves correctly
    - Walks arrays: ["${A}", "${B}"] resolves each element
    - Non-string values (numbers, booleans, null) pass through unchanged
    - Template map uses dot-bracket path notation: "tests[0].baseUrl"

    Bare unset var is error:
    - interpolateConfig({key: "${UNSET}"}, {}) => errors includes "UNSET: not set"

    Escape hatch:
    - interpolateConfig({key: "$${VAR}"}, {VAR: "value"}) => resolved.key === "${VAR}" (literal)

    Invalid syntax detection:
    - interpolateConfig({key: "${}"}, {}) => errors includes invalid syntax message
    - interpolateConfig({key: "${123}"}, {}) => errors includes invalid syntax message

    Partial substitution:
    - interpolateConfig({url: "https://${HOST}:${PORT}/api"}, {HOST: "localhost", PORT: "3000"}) => "https://localhost:3000/api"
    - Template map records the full original string

    Batch errors:
    - Multiple missing vars collected, not thrown one at a time
    - errors array contains all issues for single-pass reporting

    Empty string env var:
    - ${VAR} where VAR="" treated as unset (triggers error or default)
  </behavior>
  <implementation>
    Create src/config/interpolate.ts with:

    1. InterpolationResult interface: { resolved: unknown, templates: Map<string, string>, errors: string[] }

    2. interpolateConfig(obj: unknown, env?: Record<string, string | undefined>): InterpolationResult
       - Entry point. Defaults env to process.env.
       - Creates templates Map and errors array, calls deepWalk, returns result.

    3. deepWalk(value, path, templates, errors, env): unknown
       - Recursive traversal. For strings: call interpolateString. For arrays: map with index paths. For objects: iterate entries with dot paths. Others: passthrough.

    4. interpolateString(value, path, templates, errors, env): string
       - First check for invalid patterns: ${} (empty) and ${123...} (starts with digit). Push error if found.
       - Single regex: /\$\$\{([^}]*)\}|\$\{([A-Za-z_][A-Za-z0-9_]*)(?:(:[-?])([^}]*))?\}/g
         - Match group 1 (escaped): $${...} => return literal ${...}
         - Match group 2 (varName): the variable name
         - Match group 3 (modifier): :- or :?
         - Match group 4 (modValue): default value or error message
       - For each match:
         - If escaped (group 1 defined): return "${" + escaped + "}"
         - Look up env[varName]. If defined and non-empty: return value.
         - If modifier is ":-": return modValue (default).
         - If modifier is ":?": push "varName: modValue" to errors, return match unchanged.
         - Bare (no modifier): push "varName: not set" to errors, return match unchanged.
       - If any env ref was found (hasEnvRef flag), record templates.set(path, originalValue).

    Design notes per user decisions:
    - All string fields interpolated (no allowlist)
    - env parameter for testability (inject mock env instead of process.env)
    - Escape pattern ($${...}) MUST be first alternation in regex to avoid partial match
    - Default values and error messages cannot contain } (non-brace character class [^}]*)
    - Empty string env values treated as unset (per :- and :? POSIX convention)
  </implementation>
</feature>
