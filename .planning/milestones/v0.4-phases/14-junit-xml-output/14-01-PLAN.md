  <name>Task 1: Create xml-utils with escapeXml and stripAnsi utilities</name>
  <files>src/output/xml-utils.ts, tests/unit/output/xml-utils.test.ts</files>
  <behavior>
    - escapeXml("hello") returns "hello" (no special chars)
    - escapeXml("a & b") returns "a &amp; b"
    - escapeXml("<script>") returns "&lt;script&gt;"
    - escapeXml('say "hi"') returns "say &quot;hi&quot;"
    - escapeXml("it's") returns "it&apos;s"
    - escapeXml("a & b < c > d") handles all 5 XML chars in one string
    - escapeXml does NOT double-escape: escapeXml("&amp;") returns "&amp;amp;" (correct — escape once at interpolation point)
    - stripAnsi("\x1b[31mred\x1b[0m") returns "red"
    - stripAnsi("\x1b[1m\x1b[33mwarn\x1b[0m") returns "warn" (nested sequences)
    - stripAnsi("no ansi here") returns "no ansi here" (passthrough)
    - stripAnsi("") returns "" (empty string)
  </behavior>
  <action>
    1. Create `tests/unit/output/xml-utils.test.ts` with test cases for escapeXml and stripAnsi per the behavior spec above. Use `describe`/`test` from `bun:test`. Run tests — they must FAIL (RED).

    2. Create `src/output/xml-utils.ts` exporting two functions:
       - `escapeXml(text: string): string` — chain of 5 `.replace()` calls for `&`, `<`, `>`, `"`, `'`. IMPORTANT: `&` must be replaced FIRST to avoid double-escaping the other replacements.
       - `stripAnsi(text: string): string` — single regex replace using the well-known ansi-regex pattern: `/[\u001B\u009B][[()#;?]*(?:[0-9]{1,4}(?:;[0-9]{0,4})*)?[0-9A-ORZcf-nqry=><]/g`
