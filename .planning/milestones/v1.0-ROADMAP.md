# Roadmap: SuperGhost

## Overview

SuperGhost is built in three phases that follow strict component dependency order. Phase 1 establishes the zero-dependency foundation: config loading, CLI wiring, output formatting, and MCP subprocess lifecycle management. Phase 2 implements the core value — the AI agent loop, step caching, self-healing, and multi-provider support — as an integrated unit, since recorder, replayer, and agent are too tightly coupled to build separately. Phase 3 packages the proven tool for distribution via npm, bunx, and a standalone compiled binary.

## Phase Details

### Phase 1: Foundation
**Goal**: Users can run `superghost --config tests.yaml` and get meaningful feedback on config validity and test structure — no AI calls yet, but the full CLI scaffold is wired and process cleanup is guaranteed
**Depends on**: Nothing (first phase)
**Requirements**: CONF-01, CONF-02, CONF-03, CONF-04, CONF-05, CLI-01, CLI-02, CLI-03, CLI-04, CLI-05, INFR-01, INFR-02
**Success Criteria** (what must be TRUE):
  1. User can run `superghost --config tests.yaml` and see per-test `[RUNNING]` / `[PASS]` / `[FAIL]` output with timing
  2. User sees a clear error message and exit code 1 when the config file is missing, malformed YAML, or fails Zod validation — with line context
  3. Process exits with code 0 when all tests pass and code 1 when any test fails
  4. User sees a run summary after all tests: total, passed, failed, cached count, and wall time
  5. Killing the process with Ctrl-C or SIGTERM leaves no orphaned MCP subprocess processes behind
**Plans:** 3/3 plans complete

- [ ] 01-02-PLAN.md — ConsoleReporter, TestRunner, ProcessManager, and signal handlers

### Phase 2: Core Engine
**Goal**: Users can write a plain English test case in YAML, run it once (AI executes in real browser), run it again (cached steps replay in ~50ms), and have stale cache auto-heal when the UI changes — across all four LLM providers
**Depends on**: Phase 1
**Requirements**: AGNT-01, AGNT-02, AGNT-03, AGNT-04, AGNT-05, CACH-01, CACH-02, CACH-03, CACH-04, CACH-05, CACH-06, CACH-07, PROV-01, PROV-02, PROV-03, PROV-04, PROV-05
**Success Criteria** (what must be TRUE):
  1. User runs a browser test case for the first time and it executes in a real Chromium/Firefox/WebKit browser via Playwright MCP, reporting `[PASS]` or `[FAIL]` with an AI-generated error message on failure
  2. User writes a test case describing an HTTP call (e.g., "POST to /api/login") and it executes via curl MCP with no browser launched
  3. On a second run of a passing test, the result is `[PASS] (cache, ~50ms)` — no AI invoked
  4. User changes the app UI; the cached test auto-detects the stale cache, re-runs via AI, updates the cache on success, and deletes the cache on failure
  5. User can switch between Anthropic, OpenAI, Google Gemini, and OpenRouter by changing the `model` field in YAML — no code changes required
**Plans:** 4/4 plans complete

### Phase 3: Distribution
**Goal**: Users can install and run SuperGhost without cloning the repo — via `bunx`, global install, or a standalone binary that requires no runtime
**Depends on**: Phase 2
**Requirements**: DIST-01, DIST-02, DIST-03, DIST-04
**Success Criteria** (what must be TRUE):
  1. User can run `bunx superghost --config tests.yaml` with no prior install and tests execute correctly
  2. User can install globally via `bun install -g superghost` and invoke `superghost` from any directory
  3. User can download a standalone compiled binary and run it on a machine with no Bun or Node.js installed
**Plans:** 2/2 plans complete
