<task type="auto" tdd="true">
  <name>Task 2: Config loader with three-layer error handling</name>
  <files>src/config/loader.ts, tests/unit/config/loader.test.ts, tests/fixtures/bad-syntax.yaml</files>
  <behavior>
    - loadConfig returns parsed Config for valid YAML file
    - loadConfig throws ConfigLoadError with hint when file does not exist
    - loadConfig throws ConfigLoadError with line context when YAML syntax is invalid
    - loadConfig throws ConfigLoadError with ALL Zod issues (numbered, with field paths) when validation fails
    - ConfigLoadError has `name` property "ConfigLoadError" and preserves `cause`
  </behavior>
  <action>
    1. Create `tests/fixtures/bad-syntax.yaml`: YAML with syntax error (e.g., missing colon on a mapping line) that will trigger YAMLParseError.

    2. Write tests first (RED) in `tests/unit/config/loader.test.ts`:
       - Test: loads and returns valid config from `tests/fixtures/valid-config.yaml`
       - Test: throws ConfigLoadError with "not found" message and hint for missing file path
       - Test: throws ConfigLoadError with line/col context for `tests/fixtures/bad-syntax.yaml`
       - Test: throws ConfigLoadError listing ALL validation issues numbered for `tests/fixtures/invalid-config.yaml`
       - Test: ConfigLoadError has name "ConfigLoadError" and cause property

    3. Implement `src/config/loader.ts` (GREEN):
       - Export `ConfigLoadError` class extending Error with `name = "ConfigLoadError"` and optional `cause`
       - Export `async function loadConfig(filePath: string): Promise<Config>`
       - Layer 1 (file): Use `Bun.file(filePath).exists()` check. If not found, throw ConfigLoadError with message: `Config file not found: ${filePath}\n  Create a config file or specify a different path:\n    superghost --config <path>`
       - Layer 2 (YAML): Parse with `parseYaml(content, { prettyErrors: true })`. Catch errors, check `error.name === "YAMLParseError"`. Format with `formatYamlError()` helper that reads the source file line manually and constructs: `Invalid YAML syntax\n  ${filePath}:${line}:${col}\n  ${lineNum} | ${sourceLine}\n  ${' '.repeat(col + offset)}^ ${errorMessage}`
       - Layer 3 (Zod): `ConfigSchema.safeParse(raw)`. If `!result.success`, format ALL issues as numbered list: `Invalid config (${N} issue${s})\n  1. ${path}: ${message}\n  2. ...`
       - Return `result.data` on success
       - IMPORTANT: Check `result.success` boolean, NOT `instanceof Error` (Zod v4 pitfall)
       - Use `Bun.file(path).text()` not `node:fs/promises` (Bun-native per research)
  </action>
  <verify>
    <automated>bun test tests/unit/config/loader.test.ts</automated>
