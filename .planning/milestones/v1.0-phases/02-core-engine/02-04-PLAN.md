  <name>Task 1: TestExecutor with cache-first strategy, retry loop, and self-healing</name>
  <files>src/runner/test-executor.ts, tests/unit/runner/test-executor.test.ts</files>
  <behavior>
    Cache-first strategy:
    - When cache exists and replay succeeds, returns { status: "passed", source: "cache" } without calling agent
    - When cache exists but replay fails, calls agent (self-healing path)
    - When no cache exists, calls agent directly

    Self-healing:
    - When replay fails and agent succeeds, saves new cache and returns { status: "passed", source: "ai", selfHealed: true }
    - When replay fails and agent fails (all attempts), deletes stale cache and returns { status: "failed" }

    Retry:
    - On agent failure, retries up to maxAttempts before marking failed
    - Returns last error message from agent on final failure
    - Recursion limit exceeded (agent returns passed=false) counts as a failed attempt

    Cache miss:
    - When no cache and agent succeeds, saves cache and returns { status: "passed", source: "ai" }
    - When no cache and agent fails (all attempts), returns { status: "failed" }

    Diagnostics:
    - Failed tests include the AI agent's diagnostic message as the error field
  </behavior>
  <action>
    1. Write tests first (RED) in `tests/unit/runner/test-executor.test.ts`:
       - Use mock CacheManager, StepReplayer, and executeAgent function
       - Test cache hit + replay success -> returns passed/cache, agent NOT called
       - Test cache hit + replay fail + agent success -> returns passed/ai/selfHealed=true, cache save called
       - Test cache hit + replay fail + agent fail (1 attempt) -> returns failed, cache delete called
       - Test cache hit + replay fail + agent fail (3 attempts) -> retries 3 times then fails, cache delete called
       - Test cache miss + agent success -> returns passed/ai, cache save called
       - Test cache miss + agent fail (all attempts) -> returns failed with last error message
       - Test cache miss + agent fail (2 attempts) + agent success (3rd) -> returns passed/ai, cache save called
       - Test error message from agent is included in TestResult.error on failure
       - Each test should verify correct interactions with mocks (which methods called, how many times)

    2. Implement `src/runner/test-executor.ts` (GREEN):
       - SuperGhost's `src/runner/test-executor.ts` implementation
       - Import CacheManager, StepReplayer, executeAgent, types
       - Export class `TestExecutor`:
         Constructor accepts: `{ cacheManager: CacheManager; replayer: StepReplayer; model: any; tools: Record<string, any>; config: Pick<Config, "maxAttempts" | "recursionLimit" | "model" | "modelProvider" | "context">; globalContext?: string }`

         Method `async execute(testCase: string, baseUrl: string, testContext?: string): Promise<TestResult>`:
           a. `const start = Date.now()`
           b. Try cache: `const cached = await this.cacheManager.load(testCase, baseUrl)`
           c. If cached:
              - Replay: `const replay = await this.replayer.replay(cached.steps)`
              - If replay.success: return `{ testName: testCase, testCase, status: "passed", source: "cache", durationMs: Date.now() - start }`
              - If replay fails: fall through to agent with selfHeal=true flag
           d. Call `this.executeWithAgent(testCase, baseUrl, start, selfHeal, testContext)`

         Private method `async executeWithAgent(testCase, baseUrl, startTime, selfHeal, testContext?)`:
           a. Loop `maxAttempts` times:
              - Call `executeAgent({ model, tools, testCase, baseUrl, recursionLimit, globalContext, testContext })`
              - If result.passed:
                - Save cache: `await this.cacheManager.save(testCase, baseUrl, result.steps, { model: config.model, provider: config.modelProvider, stepCount: result.steps.length, aiMessage: result.message, durationMs: Date.now() - startTime })`
                - Return `{ testName: testCase, testCase, status: "passed", source: "ai", durationMs: Date.now() - startTime, selfHealed: selfHeal }`
              - Record lastError = result.message
           b. All attempts exhausted:
              - If selfHeal: delete stale cache `await this.cacheManager.delete(testCase, baseUrl)`
              - Return `{ testName: testCase, testCase, status: "failed", source: "ai", durationMs: Date.now() - startTime, error: lastError }`

    NOTE: The testName field is set to testCase here because TestExecutor operates on the raw test case string. The TestRunner in Phase 1 maps test.name as the display name separately.
    NOTE: Per CONTEXT.md: "Recursion limit hit: counts as a failed attempt toward maxAttempts" — this is handled naturally because executeAgent returns { passed: false } when the agent exhausts its step limit.
  </action>
