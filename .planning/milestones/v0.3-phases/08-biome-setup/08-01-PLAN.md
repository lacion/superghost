<task type="auto">
  <name>Task 1: Install Biome, create biome.json config, and add npm scripts</name>
  <files>biome.json, package.json</files>
  <action>
1. Install Biome with exact version pinning:
   ```
   bun add -D -E @biomejs/biome@2.4.6
   ```

2. Create `biome.json` at the project root with the following configuration (do NOT use `biome init` -- write it directly):
   ```json
   {
     "$schema": "https://biomejs.dev/schemas/2.4/schema.json",
     "vcs": {
       "enabled": true,
       "clientKind": "git",
       "useIgnoreFile": true
     },
     "files": {
       "includes": [
         "**",
         "!!**/node_modules",
         "!!**/dist",
         "!!**/.superghost-cache",
         "!!**/e2e"
       ]
     },
     "formatter": {
       "enabled": true,
       "indentStyle": "space",
       "indentWidth": 2,
       "lineWidth": 120,
       "lineEnding": "lf"
     },
     "javascript": {
       "formatter": {
         "quoteStyle": "double",
         "semicolons": "always",
         "trailingCommas": "all"
       }
     },
     "linter": {
       "enabled": true,
       "rules": {
         "recommended": true,
         "suspicious": {
           "noExplicitAny": "warn"
         },
         "style": {
           "useImportType": {
             "level": "on",
             "options": { "style": "inlineType" }
           }
         }
       }
     },
     "assist": {
       "actions": {
         "source": {
           "organizeImports": {
             "level": "on",
             "options": {
               "groups": [
                 [":BUN:", ":NODE:", ":PACKAGE:"],
                 ":BLANK_LINE:",
                 [":ALIAS:", ":PATH:"]
               ]
             }
           }
         }
       }
     }
   }
   ```

3. Update `package.json` scripts section -- add `lint` and `lint:fix`, update `prepublishOnly`:
   - Add: `"lint": "bunx biome check ."`
   - Add: `"lint:fix": "bunx biome check --write ."`
   - Update: `"prepublishOnly": "bun run lint && bun test && bunx tsc --noEmit"`
   Keep all existing scripts unchanged.

4. Run `bunx biome check .` (dry-run, no --write) to preview diagnostics. If any recommended rules produce errors that cannot be auto-fixed and are clearly false positives (e.g., in AI SDK callbacks or MCP schemas), disable those specific rules in biome.json under "Claude's Discretion." Do NOT disable rules that flag real issues.

5. If `package.json` or `tsconfig.json` formatting looks unreasonable after Biome processes them (significantly expanded/collapsed), add a JSON-specific override in biome.json to adjust. This is under "Claude's Discretion."
  </action>
  <verify>
    <automated>test -f biome.json && bunx biome check --help > /dev/null 2>&1 && echo "biome installed and config exists"</automated>
  </verify>
  <done>biome.json exists at project root with all locked decisions (recommended rules, warn noExplicitAny, 120 line width, double quotes, semicolons always, trailing commas all, import sorting with groups). package.json has lint, lint:fix, and updated prepublishOnly scripts. `bunx biome check --help` runs successfully.</done>
</task>

<task type="auto">
  <name>Task 2: Apply baseline formatting and create .git-blame-ignore-revs</name>
  <files>.git-blame-ignore-revs, src/**/*.ts, tests/**/*.ts, scripts/**/*.ts</files>
  <action>
1. Apply all Biome fixes to the entire codebase:
   ```
   bunx biome check --write .
   ```
   This will: format all .ts/.json files (trailing commas, line width, indentation), sort and group imports (external first, blank line, internal), convert `import type { Foo }` to inline `import { type Foo }` style, and apply any auto-fixable lint rule violations.

2. Verify zero violations remain:
   ```
   bunx biome check .
   ```
   This must exit 0. If any non-auto-fixable errors remain, add `// biome-ignore` suppression comments with a justification reason, OR disable the specific rule in biome.json if it affects more than ~5 locations (per "Claude's Discretion").

3. Verify existing tests still pass after formatting:
   ```
   bun test
   ```
   Formatting should never break functionality. If any test fails, investigate -- it means the formatting changed semantics (extremely unlikely but must verify).

4. Verify typechecking still passes:
   ```
   bunx tsc --noEmit
   ```

5. Create the formatting baseline commit with all formatted files:
   ```
   git add -A
   git commit -m "style: format codebase with biome"
   ```

6. Record the commit hash in `.git-blame-ignore-revs`:
   ```
   HASH=$(git rev-parse HEAD)
   ```
   Create `.git-blame-ignore-revs` with contents:
   ```
   # Biome initial formatting baseline
   <the actual commit hash>
   ```

7. Configure local git to use the ignore-revs file:
   ```
   git config blame.ignoreRevsFile .git-blame-ignore-revs
   ```

8. Commit the blame-ignore-revs file:
   ```
   git add .git-blame-ignore-revs
   git commit -m "chore: add .git-blame-ignore-revs for formatting commit"
   ```

9. Final verification -- run the full quality gate:
   ```
   bun run lint && bun test && bunx tsc --noEmit
   ```
   All three must pass.
  </action>
  <verify>
    <automated>bun run lint && bun test && bunx tsc --noEmit</automated>
  </verify>

Git blame locally configured to skip the formatting commit.</done>
