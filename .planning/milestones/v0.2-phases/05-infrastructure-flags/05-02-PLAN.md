  <action>
    1. Create `tests/unit/infra/preflight.test.ts` with tests for preflight logic:
       - Extract the preflight check as a standalone async function `checkBaseUrlReachable(url: string): Promise<void>` that throws on network failure and resolves on any HTTP response.
       - Test cases:
         a. "resolves for reachable URL" -- use a local HTTP server (Bun.serve) that returns 200, verify no throw
         b. "resolves for 404 response" -- server returns 404, verify no throw (reachability, not health)
         c. "resolves for 500 response" -- server returns 500, verify no throw
         d. "throws for unreachable URL" -- use a port that nothing listens on (e.g., http://127.0.0.1:19999), verify it throws
         e. "throws on timeout" -- use a server that never responds (hang the handler with a long delay), use a short timeout override for test speed
       - Export checkBaseUrlReachable from cli.ts or create a small `src/infra/preflight.ts` module for testability (preferred -- keeps cli.ts focused on wiring). If creating the module, export the function and import it in cli.ts.

    2. Create `src/infra/preflight.ts` with the `checkBaseUrlReachable` function:
       ```typescript
       export async function checkBaseUrlReachable(url: string, timeoutMs = 5000): Promise<void> {
         await fetch(url, {
           method: 'HEAD',
           signal: AbortSignal.timeout(timeoutMs),
           redirect: 'follow',
         });
       }
       ```
       This function throws on network errors (connection refused, DNS, timeout) and resolves on ANY HTTP response. The caller (cli.ts) handles the error formatting and exit.

    3. In `src/cli.ts`, after the --only filter block and before the header print:
       a. Import: `import { checkBaseUrlReachable } from './infra/preflight.ts';`
       b. Add preflight check:
       ```typescript
       // Preflight: check baseUrl reachability (only if global baseUrl configured)
       if (config.baseUrl) {
         try {
           await checkBaseUrlReachable(config.baseUrl);
         } catch {
           await Bun.write(Bun.stderr,
             `${pc.red("Error:")} baseUrl unreachable: ${config.baseUrl}\n` +
             `  Check that the server is running and the URL is correct.\n`
           );
           setTimeout(() => process.exit(2), 100);
           return;
         }
       }
       ```
       c. This MUST be placed after --only filter and before MCP init, per the locked startup order in CONTEXT.md.

    CRITICAL per CONTEXT.md decisions:
    - Only network-level failures (connection refused, DNS, timeout) cause exit 2
    - Any HTTP response (even 403, 404, 500) means the server is reachable -- do NOT check status codes
    - Use `pc.red("Error:")` prefix consistent with all other exit-2 errors
    - If no global baseUrl configured (undefined), skip preflight silently
    - The `setTimeout(() => process.exit(2), 100); return;` pattern is the established exit pattern in this codebase
  </action>
