# Requirements: SuperGhost

**Defined:** 2026-03-10
**Core Value:** Plain English test cases that execute in a real browser, with step caching that makes them fast and deterministic enough for CI/CD — no test code required.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Configuration

- [x] **CONF-01**: User can define test cases in plain English via YAML (`case: "check login is working"`)
- [x] **CONF-02**: User's YAML config is validated with Zod; clear error messages on invalid config with line context
- [x] **CONF-03**: User can configure global settings: `baseUrl`, `browser`, `headless`, `timeout`, `maxAttempts`, `model`, `modelProvider`, `cacheDir`, `recursionLimit`
- [x] **CONF-04**: User can override `baseUrl` and `timeout` per test case
- [x] **CONF-05**: Config provides sensible defaults (chromium, headless true, 60s timeout, 3 max attempts, claude model)

### CLI

- [x] **CLI-01**: User can run tests with `superghost --config tests.yaml`
- [x] **CLI-02**: User sees clear error and exit code 1 when config file is missing or malformed
- [x] **CLI-03**: User sees `[RUNNING]`, `[PASS]`, or `[FAIL]` for each test with source (`cache`/`ai`) and timing
- [x] **CLI-04**: User sees summary after all tests: total, passed, failed, cached count, and wall time
- [x] **CLI-05**: Process exits with code 0 when all tests pass, code 1 when any test fails

### Agent

- [x] **AGNT-01**: AI agent executes browser test cases by driving a real browser via Playwright MCP
- [x] **AGNT-02**: AI agent executes API test cases via curl MCP, auto-detected from test description (HTTP verbs, URL patterns)
- [x] **AGNT-03**: Each test case gets an independent browser context (no state leakage between tests)
- [x] **AGNT-04**: Agent retries failed test cases up to `maxAttempts` before marking as failed
- [x] **AGNT-05**: Failed tests include diagnostic error message from the AI agent explaining what went wrong

### Cache

- [x] **CACH-01**: Successful AI test steps are recorded and saved to `.superghost-cache/<hash>.json`
- [x] **CACH-02**: Cache key is SHA-256 hash of `(testCase + baseUrl)`, deterministic across runs
- [x] **CACH-03**: On subsequent runs, cached steps replay instantly (~50ms) without invoking the AI
- [x] **CACH-04**: When cached replay fails (UI changed), AI agent re-executes the test automatically
- [x] **CACH-05**: When AI re-execution succeeds after stale cache, cache is updated with new steps
- [x] **CACH-06**: When AI re-execution fails after stale cache, stale cache is deleted and test marked failed
- [x] **CACH-07**: Cache files are human-readable JSON with `testCase`, `baseUrl`, `steps`, timestamps

### Providers

- [x] **PROV-01**: User can use Anthropic (Claude) models as the AI provider
- [x] **PROV-02**: User can use OpenAI models as the AI provider
- [x] **PROV-03**: User can use Google Gemini models as the AI provider
- [x] **PROV-04**: User can use OpenRouter models as the AI provider
- [x] **PROV-05**: Provider is auto-inferred from model name when `modelProvider` is not specified

### Distribution

- [x] **DIST-01**: User can run SuperGhost via `bunx superghost --config tests.yaml` with no prior install
- [x] **DIST-02**: User can install globally via `bun install -g superghost`
- [x] **DIST-03**: SuperGhost is published as an npm package
- [x] **DIST-04**: User can download a standalone compiled binary (via `bun build --compile`) that runs without Bun installed

### Infrastructure

- [x] **INFR-01**: MCP subprocess processes are cleaned up on SIGINT/SIGTERM (no orphaned processes)
- [x] **INFR-02**: Agent loop has configurable step limit (`recursionLimit`) to prevent runaway LLM costs

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### CLI Enhancements

- **CLIX-01**: User can force fresh AI runs with `--no-cache` / `--clear-cache` flags
- **CLIX-02**: User can run a subset of tests with `--only <pattern>` flag
- **CLIX-03**: User can enable verbose mode (`--verbose`) showing step-level AI actions
- **CLIX-04**: User can get machine-readable results with `--output json`
- **CLIX-05**: User can skip individual tests with a `skip` flag in YAML

### Execution

- **EXEC-01**: Tests can run in parallel for faster suite execution
- **EXEC-02**: Watch mode re-runs tests on config file changes

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| GUI / Dashboard | Changes product category from CLI tool to web app; doubles scope |
| Visual regression testing | Different domain; dedicated tools (Percy, Chromatic) do it better |
| Test generation from crawling | Low signal tests that don't express intent; users should write cases |
| Cloud-hosted execution | Turns a tool into a SaaS product; CI runners cover this |
| Per-step assertions in YAML | Turns config into a programming language; AI handles assertions |
| Slack / webhook notifications | CI/CD already handles failure alerts via exit codes |
| Test tags / groups / suites | Multiple YAML files solve this without schema complexity |
