<task type="auto">
  <name>Task 1: Create ci.yml with parallel lint/typecheck/test and gate job</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` with the following structure:

**Workflow-level config:**
- `name: CI`
- Triggers: `push` on `[main]` branch, `pull_request` with types `[opened, synchronize, ready_for_review]`
- Concurrency group: `ci-${{ github.event.pull_request.number || github.ref }}` with `cancel-in-progress: true`

**Three parallel check jobs (lint, typecheck, test):**
Each job has identical setup:
- `if: ${{ !github.event.pull_request.draft }}` (skips drafts; `!null` = true so push events pass)
- `runs-on: ubuntu-latest`
- Steps: checkout@v4, setup-bun@v2 with `bun-version: "1.3.x"` (do NOT add `cache: true` -- setup-bun@v2 has no such parameter; default behavior already caches), `bun install --frozen-lockfile`, then the check command

Check commands:
- lint: `bunx biome check .`
- typecheck: `bunx tsc --noEmit`
- test: `bun test`

**Gate job:**
- Job ID: `gate` (no explicit `name` -- appears as "CI / gate" in branch protection)
- `runs-on: ubuntu-latest`
- `needs: [lint, typecheck, test]`
- `if: always()` (CRITICAL: without this, gate is skipped when upstream fails, which passes branch protection by default)
- Single step "Check results" with bash that checks all three `needs.*.result == 'success'` using `!=` checks with `exit 1` on failure
- Use strict `== 'success'` comparison (not `!= 'failure'`) so cancelled/skipped jobs also fail the gate
  </action>
  <verify>
    <automated>python3 -c "import yaml; y=yaml.safe_load(open('.github/workflows/ci.yml')); jobs=y['jobs']; assert set(['lint','typecheck','test','gate']) == set(jobs.keys()), f'Expected 4 jobs, got {list(jobs.keys())}'; assert jobs['gate']['needs'] == ['lint','typecheck','test']; assert 'frozen-lockfile' in str(y); print('ci.yml structure valid')" 2>/dev/null || bun -e "const fs=require('fs'); const c=fs.readFileSync('.github/workflows/ci.yml','utf8'); const checks=['needs: [lint, typecheck, test]','frozen-lockfile','if: always()','biome check','tsc --noEmit','bun test','name: CI']; const missing=checks.filter(c2=>!c.includes(c2)); if(missing.length){console.error('Missing:',missing);process.exit(1)} console.log('ci.yml validated')"</automated>
  </verify>
  <done>ci.yml exists with 4 jobs (lint, typecheck, test running in parallel + gate aggregator), all using frozen-lockfile, draft PR filtering, concurrency control, and strict gate success checking</done>
